# 仙窍扩展版本发布前文档闸门（Task19）

## 置顶声明（发布前必须确认）

**本版本与旧存档不兼容。**  
**不提供迁移工具。**

本次发布属于结构性扩展升级，若直接在旧存档上覆盖运行，可能出现数据错配、机制异常或进度不可逆偏差。运营与测试必须将此声明作为首要对外口径，不得弱化或省略。

## 升级建议

1. **将本版本视为新档版本**，建议使用全新世界或独立测试存档进行部署。
2. 生产环境升级前，先在预发环境执行以下命令，确认构建与规范门禁通过：

```bash
./gradlew --no-daemon --rerun-tasks build
./gradlew --no-daemon --rerun-tasks checkstyleMain
./gradlew --no-daemon runGameTestServer
```

3. 对外发布说明中，必须明确“旧存档不可直接升级、且无迁移工具”。
4. 升级窗口建议安排在低峰时段，并预留验证与回退时间。

## 回滚建议

1. 发布前先做完整备份（世界存档、配置、服务端运行参数）。
2. 若升级后出现异常，**不要继续写入问题存档**，应立即停服并回退到升级前备份。
3. 回滚后重新执行门禁命令，确认基础构建链路可用，再决定是否再次发布。
4. 若已对外公告升级，回滚公告中应同步说明“兼容性限制与后续处理计划”。

## 风险提示

- 兼容性风险：旧存档在新版本中可能发生不可预期行为。
- 数据风险：跨版本直接覆盖可能导致进度丢失、状态错位或逻辑异常。
- 运营风险：若未提前公告不兼容策略，容易造成用户预期偏差与反馈集中。
- 恢复风险：无迁移工具意味着问题存档无法自动修复，主要依赖备份回退。

```bash
./gradlew --no-daemon --rerun-tasks build
./gradlew --no-daemon --rerun-tasks checkstyleMain
./gradlew --no-daemon runGameTestServer
```
