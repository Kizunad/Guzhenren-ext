关于实体破坏方块的方法探索：

1. **level.destroyBlock(pos, dropItems, entity)** - 当前使用的方法
   - 参数：位置、是否掉落、破坏者实体
   - 这已经是正确的API，entity参数会影响掉落物的计算

2. **level.removeBlock(pos, isMoving)** - 更底层的方法
   - 只移除方块，不处理掉落
   - 不推荐用于正常的破坏逻辑

3. **Mob破坏方块相关的方法**：
   - Zombie等Mob可以破门，但这是通过Goal系统（BreakDoorGoal）实现的
   - 没有通用的Mob.breakBlock()方法
   - 当前使用的 destroyBlock(pos, true, entity) 已经是最合适的API

**结论**：当前的实现已经正确。`level.destroyBlock(blockPos, true, entity)` 会：
- 触发方块的破坏逻辑
- 根据entity计算掉落物（考虑工具、附魔等）
- 生成ItemEntity到世界中

无需修改。
